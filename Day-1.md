**Red Team (Offensive Security)**

* Simulates attacks
* Finds vulnerabilities
* Performs security assessments
* Common role: Penetration Tester

**Blue Team (Defensive Security)**

* Detects attacks
* Monitors systems
* Investigates alerts
* Responds to incidents

Common roles include:

* SOC Analyst
* Incident Responder
* Malware Analyst
* Threat Hunter

I also learned that strong foundational knowledge in networking, operating systems, and security principles is valuable regardless of specialization.

### Real-World Relevance

In a real organization:

* Penetration Testers identify weaknesses before attackers do.
* Security Analysts monitor alerts and suspicious activity.
* Incident Responders investigate security breaches.
* Malware Analysts examine malicious software.

All of these roles work together to improve an organization's security posture.

## 🛡️ SOC Analyst L1: Mindset

### Tricky Connection

A beginner often asks:

> "How can I hack into a system?"

A SOC Analyst asks:

> "How would I detect someone trying to do that?"

This is the mindset shift that stood out to me.

For example:

* Red Team learns how phishing works.

* Blue Team learns how to detect phishing attempts.

* Red Team exploits vulnerabilities.

* Blue Team identifies signs of exploitation.

Understanding both perspectives helps a SOC Analyst investigate alerts more effectively.

## Critical Thinking

### Question

A penetration testing report identifies a critical vulnerability in a company web application. Two weeks later, your SOC receives alerts indicating unusual traffic targeting the same application.

As a SOC Analyst L1, what would be your first steps?

### Answer Concept

I would:

1. Review the alert details.
2. Verify whether the traffic matches known exploitation attempts.
3. Check source IP addresses and request patterns.
4. Review web server logs for suspicious activity.
5. Determine whether the vulnerability has already been patched.
6. Escalate the event according to incident response procedures if malicious activity is confirmed.

The goal is to collect evidence, validate the threat, and understand whether the organization is experiencing an actual attack or normal activity.
